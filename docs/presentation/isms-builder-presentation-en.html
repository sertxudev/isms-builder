👥 RBAC & Multi-Function Users

The system strictly separates access rights (RBAC rank) from organizational functions — one person can hold multiple functions simultaneously.

RBAC RoleRankTypical Function
reader / revision1Employee · Internal Audit ⚖
editor / dept_head / qmb2Department head, QMO, policy author
contentowner · DPO ⚖ / auditor3CISO, DPO, compliance officer
admin4System administrator
⚖ STATUTORY INDEPENDENCE
DPO / Data Protection Officer — must not receive instructions and is protected against dismissal.
Legal basis: GDPR Art. 38(3)
Internal Audit — reports directly to the board, instruction-independent from line management.
Legal basis: IIA Standard 1100
In the system: role revision has rank 1 (read-only) — can only review, not modify. DPO gets function dso independent of RBAC rank.

Go-Live Checklist

  • Change admin password immediately
  • Configure SMTP (Admin → Organisation)
  • Set JWT_SECRET (32+ random chars)
  • Enable SSL (SSL_CERT_FILE + SSL_KEY_FILE)
  • Switch to SQLite: STORAGE_BACKEND=sqlite
  • Enforce 2FA org-wide (Admin → Security)
  • Configure CISO/DPO email recipients
  • Import your SoA control applicability
Preserved after reset: SoA controls (313), dropdown lists, org settings
Cleared: Templates, risks, GDPR, assets, BCM, legal, training, governance, suppliers, guidance, audit log, goals

📧 Email Notifications & Alerts

🛡 CISO Digest (daily)

  • Risks Score ≥ 10 (critical/high)
  • BCM plan tests in ≤ 14 days
  • Supplier audits overdue
  • Recipients: users with function ciso

🔒 DPO Digest (daily)

  • DSAR deadlines in ≤ 3 days
  • GDPR incidents open > 48h
  • Recipients: users with function dso
  • Fallback: CISO recipients

⚙ Admin Digest (daily)

  • Contracts expiring in ≤ 30 days
  • Template reviews in ≤ 14 days
  • Overdue trainings
  • Recipients: function admin_notify
Configuration: SMTP credentials either via .env or directly in Admin Panel (effective without server restart).
Dual role: CISO + DPO in one person → gets both digests automatically. No manual configuration needed.

Quick Start (Docker)

# Set JWT_SECRET:
